les principes SOLID
Les principes SOLID sont cinq concepts clés en programmation orientée objet qui visent à rendre le code plus compréhensible, flexible et maintenable. Voici un résumé de chacun de ces principes :

1. S - Single Responsibility Principle (SRP)
Chaque classe doit avoir une seule responsabilité et ne doit vous perturber que pour cette responsabilité. Cela signifie qu'une classe ne doit pas avoir plusieurs raisons de change.

2. O - Open/Closed Principle (OCP)
Les entités de logiciel (classes, modules, fonctions) doivent être ouvertes à l'extension mais fermées à la modification. Cela signifie qu'il devrait être possible d'ajouter de nouvelles fonctionnalités sans modifier le code existant.

3. L - Liskov Substitution Principle (LSP)
Les objets d'une classe dérivée doivent pouvoir remplacer les objets de la classe de base sans perturber le fonctionnement du programme. En d'autres termes les sous-types doivent être substituables à leurs types de base.

4. I - Interface Segregation Principle (ISP)
Les clients ne doivent pas être forcés d’utiliser des interfaces qu’ils n’utilisent pas. Il vaut mieux avoir plusieurs interfaces spécifiques plutôt qu'une seule interface générale.

5. D - Dependency Inversion Principle (DIP)
Les modules de haut niveau ne doivent pas dépendre des modules de bas niveau. Les deux doivent dépendre d'abstractions (interfaces). plus les détails doivent dépendre des abstractions et non l'invers
